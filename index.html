  1. Project Summary

Across all levels of baseball, there is a widespread issue in the rapidly rising rate of Ulnar Collateral Ligament (UCL) Injuries. The UCL is a ligament in the elbow that can tear with use, as a common surgery currently to solve major injuries is called Tommy John surgery.  This rise in UCL injuries has been hypothesized by some to be correlated with the increase in pitch spin rate, velocity, and curveball use at all levels.

        Our project will use classification to predict if a player is at risk for UCL injury.  We will break it up into a long-term and short-term experiment.  The long-term experiment takes season long metrics and uses different methods, such as K-Nearest Neighbors, Logistic Regression, a Multi-Layer Perceptron, and a Convolutional Neural Network to predict injury risk, comparing each of the methods.  The short-term experiment on the other hand will look at game-by-game changes in metrics, utilizing a Recurrent Neural Network to identify when a pitcher is at risk for injury.

  1. Related Work

        Our project has never really been done before, however there are two different studies that are similar.  The first[1], done in 2016, uses similar metrics and features to our project, but instead uses standard regression rather than Machine Learned based classification.  The date is important, as in 2016 Statcast, the tracker for underlying metrics like spin rate and release position, just released and was pretty new. The next study[2] was a Machine Learning study like ours and is far more recent, but it tends to look at more general stats and not specifically underlying metrics like ours.

  1. Data Sets

For our long term data, we used a couple sources.  The first for an injury log for labels was a spreadsheet with history of all UCL injuries, as well as Tommy John surgeries that have taken place[3].  We compiled all the different types of UCL injuries from this spreadsheet into one dataset that served as our labels for our data.

Next, we compiled the season long data.  We primarily used the Python API pybaseball[4], which makes calls to pull data from two main websites, Fangraphs[5] and Baseball Savant[6].  Fangraphs has all the season-long statistics any fan would need, including any counting stat, calculated stats, and even some season-long aggregates of underlying metrics.  Baseball Savant compiles all the underlying metrics in game-by-game logs, as well as season long averages which we used in the long-term experiment.

After compiling the data, we put all the datasets together and calculated averages and maxes for metrics like pitch velocity and spin rate, allowing us to the drop any null values and complete any other preprocessing for our data.  This created a dataset with over 3000 different cases spanning the past 5 seasons, with 17 different features being looked at.

The game-by-game data similarly utilized Baseball Savant, but in a far more convoluted way.  As mentioned earlier, Baseball Savant can output metrics for each game, but it outputs as so, only showing pitch-by-pitch data:

        Our actual algorithm will be mentioned more in part 4), but for each game in the season we aggregated metric totals for each pitch for each pitcher, averaged out the metrics, and then after compiling everything computed the differences in each metric between games.

        Our final dataset for the short-term dataset is a set of every pitcher over two years of game history and the differences between every game they pitched in those two years, where our series is essentially the dataframe sequence of differences.  One limitation with this data though is that some data points couldn’t compile from Baseball Savant, meaning out data is pretty limited in the number of features being used.

  1. Technical Approach

Our entire approach, from data compilation to validation, has its algorithms, so let’s start from the start in the long-term experiment.  The UCL injuries table is merged with the different injuries while ensuring no duplicates (UCL injuries where a Tommy John surgery also happened) appear.  Each row is also given an injury year, which returns the current year if the month is April or later, and otherwise, the previous year.  This is because an injury or surgery from last year could be delayed and not occur in the offseason when players rest.  We then took season-long data from Fangraphs, as well as spin rate averages from Baseball Savant, merged them into one table, added the UCL injuries table as labels for the data, and the computed max and averages for velocity and spin rate, allowing us to have a consistent table without any nulls.

The short-term experiment is a much more complex process.  To start, a list of game codes is created using a web API to get all of them in a given season.  For each game, we get the pitch-by-pitch data, as nested dictionaries are used to log the aggregates of each metric for each pitch for each pitcher in the game.  These aggregates are then averaged for the game and stored in the log of pitchers and respective tables of their game-by-game pitch data.  After the averages are taken for every game, the UCL injuries are added as labels, where 1 is if a player got hurt that season and 0 is otherwise.   We then computed the difference in metrics for each pitch between each game, as that was the data to be used.

Model Algorithms used were Multi-Layer Perceptrons (MLP), Logistic Regression, K-Nearest Neighbors (KNN), Convolutional Neural Networks (CNN) and Recurrent Neural Networks (RNN). We used MLPs for long-term injury prediction. As feedforward neural networks, MLPs have one or more hidden layers that enable them to capture complex relationships in the data. This characteristic made them suitable for our project, where understanding intricate patterns was essential. We employed logistic regression for binary classification tasks to estimate the probability of UCL injuries. This model was effective in providing a statistical approach to injury prediction. KNN's simplicity and effectiveness in classification tasks led us to use it in our project. It classifies new cases based on a similarity measure, making it a valuable tool for injury prediction where patterns can be nuanced. While CNNs are predominantly used in image processing, their ability to capture spatial hierarchies in data through convolutional filters made them a good choice for sequential data in our project. The sequential nature of our data on short-term injury prediction made RNNs an ideal choice. Their internal memory for retaining information from previous inputs is a key feature that we leveraged.

To validate our models, we adopted a K-fold cross-validation strategy. This method is essential in assessing a model's performance, as it involves dividing the dataset into several subsets and using each in turn for validation. Such an approach is crucial for understanding how well a model generalizes to new, unseen data.

        Regarding software, we primarily used TensorFlow and Keras to design and train our neural network models (MLP, CNN, RNN). We relied on sci-kit-learn for more traditional machine learning models like Logistic Regression and KNN, data preprocessing and cross-validation techniques.

Our journey included exploring various techniques to make it into the final implementation. Initially, we experimented with feature selection algorithms to identify the most relevant predictors but later pivoted to deep learning models that implicitly determine feature importance. Additionally, we considered time series analysis for injury prediction based on chronological data but eventually decided against it due to the complexity involved.

  1. Experiments and Evaluation

        Our project's experimental framework was designed with precision to evaluate the performance of various machine learning models in predicting UCL injuries in baseball pitchers. Recognizing the complexity and multifaceted nature of injury prediction, we segmented our data into two distinct categories: long-term and short-term data. This bifurcation was instrumental in tailoring specific models to the unique characteristics of each data type.

        The long-term data provided a comprehensive view of a pitcher's career and physical exertion over time. This included age, games played (G), games started (GS), and complete games (CG). These variables gave us insights into the player's experience and workload, critical factors in injury risk. Innings pitched (IP) and pitches thrown quantified the amount of time and effort a player puts into games, directly related to wear and tear. We looked at the strikeout-to-walk ratio (K/BB), fastball percentage (FB%), strike zone percentage (Zone%), first-strike percentage (F-Strike%), strikeout percentage (K%), and walk percentage (BB%). These metrics offered a nuanced view of a pitcher's skill and strategy, which could influence injury risk. Maximum velocity (MaxVelo), average velocities (AvgMedVelo, AvgSlowVelo), and spin rates (AvgSpin) provided insight into the physical demands placed on a pitcher's arm, which are crucial in assessing UCL injury risk.

        The short-term data captured the nuances of each pitch, offering a granular view of a pitcher's technique and its immediate implications. This data included more pitch-by-pitch analysis, such as game date, injury status, release speed, and release positions (x and y), allowing us to analyze each pitch's circumstances. This level of detail was crucial in understanding the immediate factors that could contribute to an injury.

        To comprehensively evaluate the performance of our machine learning models, we employed a multi-faceted approach that used test accuracy, cross-validation, and classification reports. This fundamental metric provided a straightforward assessment of each model's performance in predicting UCL injuries. By employing cross-validation, we mitigated the risk of overfitting and gained a more reliable estimate of each model's performance across different subsets of the data. This approach was particularly important given the relatively rare occurrence of UCL injuries. We went beyond test accuracy and delved into each model's precision, recall, and F1 scores. These metrics were critical in understanding how well our models could identify actual injury cases (true positives) and distinguish them from non-injuries (true negatives), which is especially important for imbalanced datasets like ours.

  1. Discussion and Conclusion

This project explored the complexities of sports injury prediction, particularly pitcher UCL injuries. We've learned the strengths and weaknesses of machine learning algorithms in real-world applications from our work. Working with KNN, Logistic Regression, MLP, CNN, and RNN helped us understand their pros and cons. While simple, the KNN and Logistic Regression models struggled with imbalanced data, as shown by their poor minority class prediction (injured cases). Complex MLP and CNN models were better at capturing nuanced data patterns, but they required more computational power and data. RNN model accuracy on short-term data was surprising. We expected good performance, but over 98% accuracy exceeded our expectations. Granular, pitch-by-pitch data could predict injuries. In KNN and Logistic Regression models, predicting rare events like injuries was harder than expected. We found it difficult to handle imbalanced datasets, a common problem in injury prediction where injury rates are low. Present models favor the majority class, resulting in low injury detection sensitivity. In contexts where detailed data is scarce, the models' dependence on data quality and granularity is a major bottleneck.

In the future, we could try an expand our data range a bit further, which in tern will help train the models further and make them more capable at predicting any given pitcher.  We can also then try and improve upon the number of features in the short-term data, again resulting in a better model that’s more effective at predicting UCL injuries.  We can even then try and improve the short-term model itself to try and predict at a given date when the exact injury happens so that it can be used to its full purpose.

Expanding further, this can evolve just past classification, as trends can be established as to which features/factors play the biggest impact in pitcher injuries.  This would involve more proper regression, as well as the establishment of association.

As more metrics become available in the future, we could begin to look into position player UCL injuries, as it’s starting to become a rising trend in all players, not just pitchers.  This would require more metrics to be tracked, as the current metrics that can be found aren’t sufficient enough, and there are a couple cases that would qualify as outliers (Shohei Ohtani, Phillies Designated Hitter Bryce Harper).
